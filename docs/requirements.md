# 新アプリ要件

## 目的

- Windows 上で動く個人利用向けの VRChat アバター管理ツールを作り直す
- アバター一覧を取得し、素早く探しやすくする
- 認証情報を安全に扱う
- 保守しやすい構造にする
- アバター切替は将来的に API 直接操作ではなく OSC ベースにする

## 想定ユーザー

- 主利用者は作者本人
- 今後、少人数に共有できる設計にしてもよいが、初期段階では個人利用優先

## 対象プラットフォーム

- Windows 専用

## MVP 機能要件

- VRChat へのログイン
- 2FA 対応
- 認証状態の確認
- 自分のアバター一覧取得
- サムネイル付き一覧表示
- 名前・説明文ベースの絞り込み
- ローカルタグによる絞り込み
- 一覧キャッシュ
- アバター詳細の再取得
- OSC 経由のアバター切替

## 追加候補

- ランダム選択
- お気に入り
- 更新日時表示
- ソート条件追加
- キャッシュ更新日時の表示

## 非機能要件

### セキュリティ

- 認証トークンを平文ファイルや `localStorage` に保存しない
- 可能なら Windows Credential Manager か同等の OS 秘密情報ストアを使う
- API 通信層と秘密情報保存層を UI から分離する

### 保守性

- 機能ごとに責務分離する
- API 応答の型を明示する
- 状態管理を 1 画面へ密集させない

### UX

- 旧版より視認性を上げる
- 主要操作を少ないクリック数で実行できる
- 一覧表示中の検索体験を優先する

### パフォーマンス

- 一覧のローカルキャッシュを使う
- サムネイル表示で体感速度を落とさない

## 制約・前提

- VRChat の認証とアバター取得は API を利用する
- アバター切替は OSC の実現方式を事前確認する必要がある
- OSC 側で直接アバター ID を指定できない場合、代替方式を設計する必要がある

## 今回の優先順位

1. セキュアな認証情報管理
2. 一覧取得と検索体験
3. 保守しやすい構成
4. 見た目の改善
5. OSC 切替

## 今後の確認事項

- VRChat 側で実現可能な OSC 切替仕様
- 永続化したいデータの範囲
- オフライン時の扱い
- キャッシュ失効ルール
